CHALLENGE EFX!

Esta es una prueba con la cual se valorara para la posicion de programador java

Puntos
1. CREAR UN SCRIPT CON 3 TABLAS, UNA LLAMADA CLIENTES, CONTENIENDO LOS SIGUIENTES CAMPOS: IdCliente, NombreCliente, FechaCreacion y Estado. LA OTRA TABLA LLAMADA PRODUCTOS, CON LOS SIGUIENTES CAMPOS: IdProducto, NombreProducto, FechaCreacion y Estado. ( LOS TIPOS DE LOS CAMPOS QUEDAN A SU CONSIDERACIÓN. ADEMÁS, LA TERCERA TABLA DEBE SER LA TABLA DE RELACIÓN ENTRE CLIENTES Y PRODUCTOS. USTED ELIGE EL NOMBRE Y SUS CAMPOS RESPECTIVOS).
2. SOLVENTAR LOS PROBLEMAS EN EL ARCHIVO POM DEL PROYECTO (SE PERMITE AGREGAR O MODIFICAR, PERO NO ELIMINAR NI COMENTAR CUALQUIER DEPENDENCIA).
3. CARGAR AL INICIAR LA APLICACIÓN EL ARCHIVO DE PROPERTIES LLAMADO “challenge-dev.properties” QUE DEBERÁ ESTAR UBICADO EN “resources/properties/” (CREAR DICHO ARCHIVO CON LAS PROPIEDADES NECESARIAS PARA EL FUNCIONAMIENTO DE LA APLICACIÓN).
4. REALIZAR LA CONFIGURACIÓN NECESARIA PARA LA BASE DE DATOS EN EL ARCHIVO DBConf QUE YA EXISTE EN EL PROYECTO
5. REPARAR LA QUERY QUE SE ENCUENTRA EN EL ARCHIVO hqlQuerys.xml DEL PROYECTO, PARA QUE SEA FUNCIONAL EN LA BÚSQUEDA DE PRODUCTOS ACTIVOS DE UN CLIENTE ESPECÍFICO. DEBE CONSERVAR EL PASO DE PARÁMETROS DEL ID DE CLIENTE Y ESTADO DEL PRODUCTO
6. HACER QUE EL MÉTODO findAllActiveProdByCliente EN LA CLASE ProductoDaoImpl RETORNE LA LISTA DE PRODUCTOS DEL CLIENTE ESPECIFICADO
7. AGREGAR UN TEXTO INFORMATIVO AL index.html Y QUE SE MUESTRE AL CARGAR LA APLICACIÓN. DEBERÁ CONFIGURAR EN LA CLASE SecurityConf LO NECESARIO PARA QUE SPRING SECURITY EVITE QUE SE PUEDA ACCEDER A OTRA DIRECCIÓN QUE NO SEA EL INDEX EN EL PATH “/”.
8. LA APLICACIÓN DEBE PODER EJECUTARSE EN TOMCAT STANDALONE Y NO EN EL EMBED QUE TRAE POR DEFECTO SPRING BOOT.
9. ACTUALIZAR SPRING BOOT A LA VERSIÓN 2.2.4 Y REALIZAR LOS CAMBIOS NECESARIOS PARA QUE FUNCIONE CON OPENJDK 14.
